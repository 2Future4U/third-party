# Listas de comprobación de extremo a extremo
{: #checklist}

Utilice las siguientes listas de comprobación para realizar un seguimiento de todas las tareas necesarias para definir, desarrollar y publicar el servicio de facturación integrado.
{:shortdesc}

## Desarrollo del servicio
{: #develop}

| Tarea | Subtareas | Descripción | Entorno |
|------| ----------| ------------|-----|
| Obtenga más información sobre la especificación de Open Service Broker versión 2.12. | ☐ He leído las especificaciones de Open Service Broker y comprendo que debo desarrollar mi propio Open Service Broker. <br><br>  |  Los intermediarios de servicio gestionan el ciclo de vida de los servicios. La plataforma {{site.data.keyword.Bluemix_notm}} interactúa con los OSB (Open Service Brokers) para suministrar y gestionar instancias de servicio y enlaces de servicio. Para obtener más detalles, consulte [Desarrollo y alojamiento de intermediarios de servicio](/docs/third-party/cis3-broker.html#step3-osb). | Documentación |
| Consulte los ejemplos de intermediarios de {{site.data.keyword.Bluemix_notm}}. | ☐ He clonado el repositorio, he visto los ejemplos de intermediarios y estoy listo para utilizar estos ejemplos para empezar el desarrollo. <br><br>  |  [https://github.com/IBM/sample-resource-service-brokers](https://github.com/IBM/sample-resource-service-brokers){: new_window} ![Icono de enlace externo](../icons/launch-glyph.svg "Icono de enlace externo") | Ejemplos de código |
| Consulte la documentación de API de Open Service Broker de {{site.data.keyword.Bluemix_notm}}. | ☐ Comprendo que hay varios puntos finales obligatorios a los que debe dar soporte el código de mi Open Service Broker. <br><br>  ☐ Comprendo que, si mi servicio se puede enlazar a apps en {{site.data.keyword.Bluemix_notm}}, debe poder devolver los puntos finales de la API y las credenciales a los usuarios de mi servicio. <br><br> ☐ Comprendo que {{site.data.keyword.Bluemix_notm}} ha definido puntos finales de API ampliados que permiten inhabilitar y volver a habilitar las instancias de servicio. |  {{site.data.keyword.Bluemix_notm}} Open Service Broker amplía la especificación Open Service Broker 2.12. [Obtenga más información](/docs/third-party/cis3-broker.html#docs) sobre los puntos finales obligatorios que debe utilizar el intermediario de servicio. | Documentación |
| Utilice los metadatos del catálogo para empezar a desarrollar el intermediario de servicio. | ☐ Comprendo que algunos de los metadatos que he proporcionado en la consola de gestión de recursos se deben exportar y suministrar en mi Open Service Broker.  <br><br>  ☐ He añadido los GUID necesarios y otros valores obligatorios que aparecen en el archivo `catalog.json` a la matriz de servicios de mi Open Service Broker de ejemplo. | Después de descargar un archivo `catalog.json` desde la consola de gestión de recursos, puede utilizarlo para informar sobre el desarrollo de Open Service Broker. Para obtener más detalles, consulte [los pasos para utilizar los metadatos exportados en el desarrollo de guías](/docs/third-party/cis3-broker.html#use-metadata). | Entorno de desarrollo y documentación |
| Aloje el Open Service Broker. | ☐ Comprendo que la ubicación alojada de mi intermediario de servicio debe seguir el protocolo Transport Layer Security (TLS) versión 1.2. <br><br> | El intermediario de servicio debe estar alojado como parte de una app que pueda responder a las llamadas de API REST. Y su ubicación alojada debe cumplir las directrices de seguridad de {{site.data.keyword.Bluemix_notm}}. Para obtener más detalles, consulte [los pasos para alojar el intermediario de servicio](/docs/third-party/cis3-broker.html#host). | Entorno de desarrollo y documentación |
| Pruebe el Open Service Broker alojado. | ☐ He validado cada uno de los puntos finales de la API a los que da soporte mi intermediario de servicio ejecutando mandatos curl sobre el intermediario de servicio alojado. <br><br> | Valide el intermediario de servicio ejecutando mandatos curl sobre los distintos puntos finales que va a habilitar. Para obtener más detalles, consulte [los pasos para probar el intermediario de servicio](/docs/third-party/cis3-broker.html#test).| Entorno de desarrollo y documentación |
| Obtenga y defina el URI de redirección de IAM. | ☐ He obtenido correctamente el URI de redirección de IAM utilizando la ubicación alojada de mi Open Service Broker y alguna información de autenticación. <br><br> ☐ He completado correctamente la página Gestión de acceso especificando mi URI de redirección y estableciendo correctamente mi ID de cliente. <br><br> | Cuando define el servicio en la consola de gestión de recursos, genera un ID de cliente, pero tenga en cuenta que probablemente no tenía un URI de redirección en ese momento. Para obtener más detalles, consulte [los pasos para obtener el URI de redirección](/docs/third-party/cis5-iam.html#redirect-uri). | Entorno de desarrollo y la consola de gestión de recursos |
| Desarrolle el flujo de Open Authorization (OAuth) para la autenticación. | ☐ Comprendo que IAM da soporte a Open ID Connect (OIDC). <br><br> ☐ He descubierto mi punto final regional de IAM y he redirigido correctamente a los usuarios desde `dashboard_url` a mi URI de redirección ensamblada, y he guardado el valor `access_token` de usuario devuelto en mi respuesta para que se pueda utilizar durante la autenticación. <br><br> | Los usuarios que visiten el panel de control de servicio deben estar correctamente autenticados. Desarrolle la autenticación y la autorización basadas en señales OAuth mediante IAM. Para obtener más detalles, consulte [los pasos para desarrollar el flujo de OAuth](/docs/third-party/cis5-iam.html#oauth). | Entorno de desarrollo y documentación|
| Valide la autorización de usuario. | ☐ Puedo obtener una señal de acceso para una clave de API. <br><br> ☐ Puedo validar la autorización para los usuarios para la instancia de servicio. <br><br> | Después de desarrollar el flujo de OAuth para la autenticación, debe validar que los usuarios estén correctamente autorizados, asegurándose de que los usuarios del servicio puedan acceder al panel de control del servicio. Para obtener más detalles, consulte [los pasos para validar](/docs/third-party/cis5-iam.html#validate). | Entorno de desarrollo y documentación |
